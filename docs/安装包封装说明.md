# 安装包封装说明

本文说明如何在 Windows 开发机上把“文件重命名助手 v1.0”封装成 `.exe` 安装包，并把运行环境一起打包进去。

## 准备环境

需要在 Windows 11 开发机安装：

- Visual Studio 2022
- .NET 8 SDK
- Windows App SDK 开发工作负载
- Inno Setup 6

## 准备随包运行环境

把以下两个官方离线安装文件放入：

```text
installer/redistributables
```

文件名必须保持为：

```text
windowsdesktop-runtime-8.0-win-x64.exe
WindowsAppRuntimeInstall-x64.exe
```

说明：

- `windowsdesktop-runtime-8.0-win-x64.exe` 用于安装 .NET 8 Desktop Runtime
- `WindowsAppRuntimeInstall-x64.exe` 用于安装 Windows App Runtime

如果你选择自包含发布，也可以不安装 `.NET 8 Desktop Runtime`，但本项目默认采用“框架依赖发布 + 安装包携带运行环境”的方式，安装包体积更可控。

## 一键发布

在 Windows PowerShell 中执行：

```powershell
cd 文件重命名助手_v1.0\installer
.\Build-Release.ps1
```

脚本会执行：

1. 清理旧发布目录
2. 还原 NuGet 包
3. 发布 WinUI 3 应用
4. 检查随包环境文件是否存在
5. 调用 Inno Setup 生成 `.exe` 安装包

成功后输出目录为：

```text
artifacts/installer
```

安装包文件名：

```text
文件重命名助手_1.0.0_Setup.exe
```

## 安装时环境检测逻辑

安装包会把 `Install-Prerequisites.ps1` 安装到程序目录，并在安装阶段执行。

检测逻辑：

```text
检查 .NET 8 Desktop Runtime
├─ 已安装：跳过
└─ 未安装：运行 windowsdesktop-runtime-8.0-win-x64.exe /install /quiet /norestart

检查 Windows App Runtime
├─ 已安装：跳过
└─ 未安装：运行 WindowsAppRuntimeInstall-x64.exe --quiet
```

## 手动编译主程序

如果只想发布主程序，不生成安装包：

```powershell
dotnet publish ..\src\FileRenameAssistant\FileRenameAssistant.csproj -c Release -r win-x64 --self-contained false -o ..\artifacts\publish\win-x64
```

## 常见问题

### 运行提示缺少 Windows App Runtime

确认 `installer/redistributables/WindowsAppRuntimeInstall-x64.exe` 已放入，并重新生成安装包。

### 运行提示缺少 .NET

确认 `installer/redistributables/windowsdesktop-runtime-8.0-win-x64.exe` 已放入，并重新生成安装包。

### Inno Setup 命令不可用

安装 Inno Setup 6 后，确认 `ISCC.exe` 位于：

```text
C:\Program Files (x86)\Inno Setup 6\ISCC.exe
```
