# 打包失败修复说明

你截图中的错误表现为：

```text
'icy' 不是内部或外部命令
'1' 不是内部或外部命令
'o' 不是内部或外部命令
```

这不是 WinUI 编译错误，也不是 Inno Setup 打包错误，而是 Windows CMD 在执行 `.bat` 文件时把脚本内容按错误编码或错误格式解析了，导致正常命令被识别成乱码命令。

## 已修复内容

v2.0 的安装脚本已做以下调整：

| 文件 | 修改 |
|---|---|
| `installer/一键生成安装包.bat` | 改成极简转发脚本，只调用英文入口 |
| `installer/Build-Installer.cmd` | 新增纯 ASCII 入口，避免中文和编码问题 |
| `installer/Build-Release.ps1` | 改成纯 ASCII 内容，并增强 dotnet、Inno Setup、运行环境文件检测 |
| `installer/FileRenameAssistant.iss` | 改成纯 ASCII 安装脚本，输出 `FileRenameAssistant_1.0.0_Setup.exe` |
| `installer/Install-Prerequisites.ps1` | 改成纯 ASCII 错误提示，避免安装阶段 PowerShell 编码问题 |

## 推荐运行方式

优先双击这个文件：

```text
installer/Build-Installer.cmd
```

如果仍想使用中文入口，也可以双击：

```text
installer/一键生成安装包.bat
```

它现在只会转发到 `Build-Installer.cmd`。

## 当前脚本会检查什么

打包脚本会按顺序检查：

1. 是否安装 `.NET 8 SDK`
2. 是否安装 `Inno Setup 6`
3. 是否存在 `.NET 8 Desktop Runtime` 离线安装包
4. 是否存在 `Windows App Runtime` 离线安装包
5. 是否能成功 `dotnet restore`
6. 是否能成功 `dotnet publish`
7. 是否能生成 `文件重命名助手_1.0.0_Setup.exe`

## 需要放入的环境文件

请把以下两个文件放到：

```text
installer/redistributables
```

文件名必须是：

```text
windowsdesktop-runtime-8.0-win-x64.exe
WindowsAppRuntimeInstall-x64.exe
```

这两个文件会被打包进安装器。用户安装软件时，如果本机缺少运行环境，安装器会静默安装它们。

## 生成结果

成功后安装包位置为：

```text
artifacts/installer/FileRenameAssistant_1.0.0_Setup.exe
```

## 为什么安装包文件名改成英文

你的错误日志说明当前 Windows PowerShell 会把 UTF-8 中文脚本当成本机 ANSI 编码解析，导致脚本内容变成乱码。为了让这台电脑先稳定打包，v2.0 修复版把安装相关脚本全部改成 ASCII，包括安装包输出名。

软件本体 UI 仍然是中文；这里只是安装器文件名和安装向导文字先使用英文，目的是避免编码导致打包失败。

## 2026-06-22 编译修复

如果发布阶段出现：

```text
error CS0103: 当前上下文中不存在名称“DataContext”
```

原因是 WinUI 3 的 `Window` 类型没有 WPF 那种直接可用的 `DataContext` 属性。修复方式是给窗口根 `Grid` 设置名称 `RootGrid`，然后在 `MainWindow.xaml.cs` 中使用：

```csharp
RootGrid.DataContext = _viewModel;
```

当前修复版已包含这个修改。

## 2026-06-22 SDK 版本修复

如果发布阶段出现：

```text
Microsoft.Build.Packaging.Pri.Tasks.ExpandPriContent
Could not load file or assembly ... dotnet\sdk\10.0.301\Microsoft\VisualStudio\v18.0\AppxPackage\Microsoft.Build.Packaging.Pri.Tasks.dll
```

说明当前电脑正在用 `.NET 10 SDK` 编译 WinUI 3 + Windows App SDK 项目。普通 .NET 项目通常可以用更高版本 SDK 编译低版本目标框架，但 WinUI 3 发布过程依赖 Windows App SDK 的 PRI 资源生成任务，使用 .NET 10 SDK 时可能寻找不存在的 Visual Studio v18 打包任务。

当前修复版已新增：

```text
global.json
```

用于锁定 `.NET 8 SDK`。打包脚本也会在开始阶段检查是否安装了 8.x SDK。如果只安装了 10.x SDK，会提前停止并提示安装 `.NET 8 SDK x64`。

请安装：

```text
.NET 8 SDK x64
```

安装后在命令行确认：

```text
dotnet --list-sdks
```

输出里必须至少包含一行类似：

```text
8.0.xxx [C:\Program Files\dotnet\sdk]
```

## 2026-06-22 WinUI PRI 任务修复

如果已经安装 `.NET 8 SDK`，但仍然出现：

```text
Microsoft.Build.Packaging.Pri.Tasks.ExpandPriContent
Could not load file or assembly ... Microsoft.Build.Packaging.Pri.Tasks.dll
```

说明 `dotnet publish` 仍然找不到 WinUI 3 发布需要的 Visual Studio AppxPackage/PRI 任务。WinUI 3 项目不建议只用裸 `.NET SDK` 发布，应该使用 Visual Studio 2022 的 `MSBuild.exe`。

当前修复版已把发布方式改成：

```text
Visual Studio 2022 MSBuild.exe /t:Publish
```

脚本会自动查找：

```text
C:\Program Files\Microsoft Visual Studio\2022\Community\MSBuild\Current\Bin\MSBuild.exe
C:\Program Files\Microsoft Visual Studio\2022\Professional\MSBuild\Current\Bin\MSBuild.exe
C:\Program Files\Microsoft Visual Studio\2022\Enterprise\MSBuild\Current\Bin\MSBuild.exe
C:\Program Files\Microsoft Visual Studio\2022\BuildTools\MSBuild\Current\Bin\MSBuild.exe
```

并检查这个文件是否存在：

```text
MSBuild\Microsoft\VisualStudio\v17.0\AppxPackage\Microsoft.Build.Packaging.Pri.Tasks.dll
```

如果缺少，请打开 Visual Studio Installer，修改当前 Visual Studio 安装，勾选：

```text
Windows application development
```

如果你的安装器界面是中文，通常叫：

```text
Windows 应用程序开发
```

同时建议确认已安装：

```text
.NET desktop development
Windows 10/11 SDK
Windows App SDK 相关组件
MSBuild
```

安装完成后重新运行：

```text
installer\Build-Installer.cmd
```

## 2026-06-22 PRI 路径检测修复

如果脚本提示：

```text
Visual Studio AppxPackage PRI task was not found
```

但你已经安装了 Visual Studio 2022 和 Windows 应用开发组件，可能是旧脚本把 Visual Studio 根目录推导错了。

当前修复版已修正路径计算。脚本会从：

```text
C:\Program Files\Microsoft Visual Studio\2022\Community\MSBuild\Current\Bin\MSBuild.exe
```

正确回退到：

```text
C:\Program Files\Microsoft Visual Studio\2022\Community
```

然后检查：

```text
C:\Program Files\Microsoft Visual Studio\2022\Community\MSBuild\Microsoft\VisualStudio\v17.0\AppxPackage\Microsoft.Build.Packaging.Pri.Tasks.dll
```

如果仍然找不到，脚本会打印 `MSBuild path` 和 `Expected PRI task`，你可以把这两行发给我继续定位。

## 2026-06-22 Visual Studio 18 路径修复

如果你的电脑显示：

```text
MSBuild path: C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe
Expected PRI task: ...\v17.0\AppxPackage\Microsoft.Build.Packaging.Pri.Tasks.dll
```

说明你安装的是 Visual Studio `18` 目录结构，但旧脚本只检查了 `v17.0` 任务目录。

当前修复版已改为依次检查：

```text
v18.0\AppxPackage
v17.0\AppxPackage
v16.0\AppxPackage
```

如果找到任一版本的：

```text
Microsoft.Build.Packaging.Pri.Tasks.dll
```

脚本就会继续打包。
