# 文件重命名助手 v1.0 升级版说明

v2.0 的核心目标是提升“使用感”：界面统一、操作更顺、输入即预览、批量文件场景下更流畅。

## 体验变化

### 自动刷新

v1.x 中虽然已经支持刷新预览，但用户仍容易形成“改规则后再点刷新”的操作习惯。v2.0 将规则输入、开关切换、排序切换都接入自动刷新。

用户输入一项命令，例如添加前缀：

```text
启用前缀 → 输入 2026_ → 点击下一个命令框
```

系统会自动刷新预览，不需要单独点击“刷新预览”。

### 防抖刷新

连续输入时不会每个字符都重新计算。系统会等待约 250 毫秒，如果用户停止输入或切换输入框，再刷新预览。

这样既能保持实时感，又能减少大文件列表时的卡顿。

### 统一 UI

v2.0 统一了这些视觉元素：

- 顶部命令栏
- 规则配置卡片
- 摘要卡片
- 预览列表
- 状态文案
- 底部安全提示

### 动画效果

v2.0 增加轻量动效：

- 页面内容进入动画
- 预览列表项目进入动画
- 列表变化重排动画
- 按钮悬停与按压反馈由 WinUI 控件样式统一处理

## 性能优化

| 优化项 | 说明 |
|---|---|
| 后台预览 | 预览构建在后台任务中完成，减少 UI 卡顿 |
| 防抖计算 | 连续输入合并为一次预览刷新 |
| 批量替换列表 | 计算完成后一次性更新预览集合 |
| 自然排序比较器 | 文件名包含数字时按数字大小排序 |
| 文件扫描容错 | 单个文件读取失败不会中断整体导入 |

## 新增科研能力

v2.0 增加以下能力骨架：

- 自然排序
- 日期规范化
- 文件元数据读取服务
- 图片拍摄时间读取接口
- 文件修改时间命名基础接口

完整图片 EXIF 和视频元数据读取将在后续版本中继续增强。
