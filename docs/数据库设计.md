# 数据库设计

v1.0 沿用 v0.5 的 SQLite 数据库设计。

数据库只保存规则、历史记录和文件路径，不保存文件内容。

## RuleProfiles

保存用户命名规则。

| 字段 | 类型 | 说明 |
|---|---|---|
| Id | TEXT | 规则 ID |
| Name | TEXT | 规则名称 |
| JsonConfig | TEXT | 规则配置 JSON |
| CreatedAt | TEXT | 创建时间 |
| UpdatedAt | TEXT | 更新时间 |

## OperationHistory

保存批量操作摘要。

| 字段 | 类型 | 说明 |
|---|---|---|
| Id | TEXT | 操作 ID |
| OperationType | TEXT | 操作类型 |
| Summary | TEXT | 操作摘要 |
| CreatedAt | TEXT | 操作时间 |
| Status | TEXT | 状态 |

## OperationItems

保存每个文件的改名前后路径。

| 字段 | 类型 | 说明 |
|---|---|---|
| Id | TEXT | 明细 ID |
| HistoryId | TEXT | 所属历史 ID |
| OriginalPath | TEXT | 原路径 |
| NewPath | TEXT | 新路径 |
| OriginalName | TEXT | 原文件名 |
| NewName | TEXT | 新文件名 |
| Status | TEXT | 状态 |
| ErrorMessage | TEXT | 错误信息 |
