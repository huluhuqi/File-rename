# 操作逻辑重构说明

本版本按新的产品逻辑重构了排序、重命名和导入列表处理方式。

## 操作顺序

软件执行顺序固定为：

```text
导入文件
↓
按排序条件链排序
↓
按重命名目标选择文件名或扩展名
↓
按重命名规则链依次执行
↓
生成预览
↓
执行真实重命名
```

## 排序条件链

默认提供一个排序条件：

```text
自然文件名
```

用户可以添加多个排序条件，例如：

```text
1. 文件类型/扩展名
2. 创建日期
3. 文件大小
```

列表从上到下代表优先级。可以通过“上移”“下移”调整顺序。

## 重命名目标

默认修改：

```text
文件名
```

如果要修改扩展名，需要把“修改对象”切换为：

```text
扩展名
```

文件名和扩展名共用同一套规则链。

## 重命名规则链

默认提供一种重命名方式：

```text
查找替换
```

可添加的规则包括：

- 查找替换
- 添加前缀
- 添加后缀
- 删除文字
- 自动编号
- 全大写
- 全小写
- 首字母大写
- 单词首字母大写

规则按列表从上到下依次执行。可以通过“上移”“下移”调整执行顺序。

## 大小写规则

大小写规则默认只作用于当前选择的重命名目标。

如果“修改对象”为：

```text
文件名
```

则只修改文件名大小写，不修改扩展名。

如果“修改对象”为：

```text
扩展名
```

则只修改扩展名大小写。

## 从列表移除

新增“从列表移除”按钮。

它只会把当前选中的文件从预览/重命名列表中移除，不会删除磁盘上的真实文件。
